# 자막 번역 가이드

## 기본 규칙

- **목표 언어**: 한국어 (사용자가 지정하면 변경)
- **타임코드 불변**: 타임코드는 절대 수정하지 않는다
- **원문 보존 대상**: 기술 용어, 고유명사, 브랜드명, 코드 식별자 (Codex, CLI, IDE, SDLC 등)
- **문체**: 자연스러운 구어체. 영상 자막이므로 읽기 쉬워야 한다
- **원본 파일 수정 금지**: 원본은 그대로 두고 새 파일로 생성

## 자동 생성 자막 정제

자동 생성 자막(auto-caption)은 수동 자막보다 노이즈가 많다. 번역 전에 정제한다.

- **중복 라인 제거**: 연속 cue에 동일 텍스트가 반복되면 첫 번째만 유지하고 나머지 제거
- **positioning 태그 제거**: `align:start`, `position:10%` 등 VTT positioning 속성 제거
- **빈 cue 제거**: 텍스트 없이 타임코드만 있는 cue 제거

## 필러 cue 처리

자동 생성 자막에는 의미 없는 필러가 많다. 번역 시 제거한다.

제거 대상 (cue 전체가 이것만으로 구성된 경우):
- 영어: `uh`, `um`, `uh-huh`, `hmm`, `oh`, `ah`, `huh`, `mm`
- 한국어: `어`, `음`, `아`, `흠`
- 반복/조합도 포함: `uh uh`, `um um`, `oh oh`

제거하지 않는 경우:
- 실질 내용과 함께 있는 필러 (예: `Um, so the next step is...` → 필러만 빼고 번역)
- 감탄사가 맥락상 의미 있는 경우 (놀람, 강조 등)

## 번역 구조 규칙

- **타임코드 불변**: 유의미한 cue의 타임코드는 그대로 유지
- **cue 순서 유지**: 순서 변경, 병합, 분할 금지
- **필러 제거로 cue 수는 줄어들 수 있음**: 원본과 정확히 일치할 필요 없음
- **의미 있는 cue 생략 금지**: 필러 외 내용은 빠뜨리지 않고 끝까지 처리. 품질 우선

## 번역 품질 기준

1. **직역 금지** — 영어 어순 그대로 옮기지 않는다
2. **의역/요약/생략 금지** — 의미 보존을 우선하되 원문 정보는 빠뜨리지 않는다
3. **존댓말 통일** — 기본 해요체. 강연/튜토리얼은 합니다체, 데모/발표는 해요체
4. **줄임말 허용** — 자막 특성상 짧은 표현 선호
5. **맥락 유지** — 이전/이후 cue와 맥락이 이어져야 한다

## 청크 처리 (긴 자막)

### 기준
- 500 cue 이하: 한 번에 번역
- 500 cue 초과: 200-300 cue 단위로 분할

### 청크 분할 규칙
- 문장 중간에서 자르지 않는다
- 가능하면 화자 전환이나 주제 전환 지점에서 분할
- 각 청크의 첫/끝 cue를 인접 청크와 1-2개 겹쳐 맥락 유지

### 청크 병합
- 타임코드가 동일한 cue를 중복으로 식별한다
- 겹친 cue는 후속 청크 버전을 채택 (더 많은 맥락을 본 번역)
- 타임코드 연속성 확인

## VTT → SRT 변환 규칙

### VTT 구조
```
WEBVTT

00:00:01.000 --> 00:00:04.000
첫 번째 자막

00:00:05.000 --> 00:00:08.000
두 번째 자막
```

### SRT 구조
```
1
00:00:01,000 --> 00:00:04,000
첫 번째 자막

2
00:00:05,000 --> 00:00:08,000
두 번째 자막
```

### 변환 절차
1. `WEBVTT` 헤더 및 메타데이터 줄 제거
2. 각 cue에 순번 부여 (1부터)
3. 타임코드의 밀리초 구분자 `.` → `,`
4. cue 사이 빈 줄 유지
5. 파일 인코딩: UTF-8 (BOM 없음)

### 변환 시 주의사항
- 타임코드 줄만 `.` → `,` 변환. 자막 텍스트의 마침표는 건드리지 않는다
- VTT의 positioning/alignment 태그(`align:`, `position:` 등)는 제거
- HTML 태그(`<b>`, `<i>` 등)는 유지 (SRT도 지원)

## 번역 후 검증

번역 완료 후 반드시 수행:

1. **cue 수 비교** — 원본 cue 수, 제거된 필러 cue 수, 번역본 cue 수를 보고
2. **타임코드 일치** — 첫/끝 cue의 타임코드가 원본과 동일한지 확인
3. **spot check** — 앞부분, 중간, 끝부분에서 각 2-3개 cue를 샘플링하여 확인: 원문 의미가 보존되는지, 한국어 문법이 자연스러운지, 존댓말이 통일되어 있는지
4. **구조 검증** — WEBVTT 헤더(VTT), cue 번호(SRT), 빈 줄 구조가 올바른지 확인

## 파일명 규칙

```
{영상제목}.{언어코드}.{포맷}
```

- 영상 제목: yt-dlp 출력 파일명 기반, 특수문자 제거
- 언어 코드: ISO 639-1 (ko, en, ja, zh 등)
- 예시:
  - `codex-for-software-engineers.en.vtt` (원본)
  - `codex-for-software-engineers.ko.srt` (번역본)
