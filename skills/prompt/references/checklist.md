# 다듬기 체크리스트

처방 후 최종 검증용. 놓친 게 없는지 빠르게 훑는다.

---

## 구조

- [ ] Instruction이 프롬프트 시작에 배치되어 있는가
- [ ] Instruction/Context/Input이 delimiter로 분리되어 있는가
- [ ] Output 형식(포맷, 길이, 언어)이 명시되어 있는가
- [ ] User input과 system instruction이 구조적으로 분리되어 있는가

## 표현

- [ ] 부정형 지시("~하지 마")가 긍정형 + fallback으로 변환되었는가
- [ ] 모호한 수량/정도가 수치로 구체화되었는가
- [ ] 불필요한 중복과 noise가 제거되었는가
- [ ] 핵심 지시가 충분히 강조(시작/끝 배치, 반복)되었는가
- [ ] 구체적 modifier를 활용했는가 ("thoughtful details", "step-by-step" 등 품질을 올리는 표현)
- [ ] Label, 용어에 오타가 없는가

## 기법

- [ ] 태스크 복잡도 대비 적정한 기법인가 (더 단순한 접근으로 충분하지 않은가)
- [ ] 사실 기반 작업에 "모르면 모른다고" fallback이 있는가
- [ ] Few-shot 사용 시 label 분포가 균형 잡혀 있고 순서가 random인가

## 안전

- [ ] 민감 정보(API 키, 내부 IP, 비밀번호 등)가 포함되어 있지 않은가
- [ ] 코드 생성 프롬프트인 경우, 테스트 필요성이 언급되었는가
