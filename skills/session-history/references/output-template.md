# Session History — 출력 템플릿

아래 템플릿에서 해당 세션에 내용이 있는 섹션만 포함한다:

```markdown
# {project} — {YYYY-MM-DD HH:mm}

## 요약
첫 문단만 읽어도 세션이 복원되도록 쓴다. 한 문단(3-6문장)에 다음 셋을 자연스럽게 녹인다:
- **요청** — 사용자가 원래 무엇을 묻거나 시켰는가. 의도를 뭉개지 말고 원래 질문이 드러나게.
- **맥락** — 왜 중요했는가. 어떤 문제·제약·동기에서 출발했는가.
- **결과/후속** — 어떻게 끝났는가, 다음에 이어질 것은 무엇인가.

(항목을 그대로 나열하지 말 것. 읽히는 한 문단으로 통합한다.)

## 주요 결정
- **{결정 내용}** — {근거/이유}

## 피드백 & 수정
- {사용자가 수정 지시한 내용} — {왜 수정했는지}

## 작업 내역
- [x] 완료된 작업
- [ ] 시도했으나 미완료

## 변경 사항
- `path/to/file.ts` — 변경 내용 한 줄 설명

## 후속 작업
- 다음에 이어서 할 것, 열린 질문, 주의사항
```

## 섹션 순서의 이유

섹션 순서는 의도적이다. 맨 위 "요약"은 첫 문단만으로 세션을 복원시키는 가장 비싼 부동산이고, 뒤이은 "주요 결정"·"피드백"이 "왜"의 근거를 담는다. 작업 내역과 변경 사항은 부차적 맥락이라 분량이 모자라면 가장 먼저 양보한다. 복원 우선순위: **요약 > 주요 결정 > 피드백 > 작업 내역 ≈ 변경 사항**.
